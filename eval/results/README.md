# Eval Results Guide

This folder stores machine-readable evaluation artifacts generated by `python -m eval.harness`.

## Tracked Metrics

- `citation_accuracy`: Fraction of extracted claims marked verified by the verifier.
- `completeness`: Fraction of planner sub-questions addressed in the final answer.
- `hallucination_rate`: Fraction of extracted claims not supported by retrieved evidence.
- `uncertainty_reported` (unanswerable only): Whether the answer explicitly signals missing/insufficient evidence.
- `tool_calls`: Approximate end-to-end API call count for efficiency/cost proxy.

## Experiment Configurations

- `no_plan_no_verify`: Uses original question directly and skips claim verification.
- `plan_no_verify`: Adds decomposition into sub-questions but skips claim verification.
- `plan_verify`: Full pipeline with planning, retrieval/compression, synthesis, and claim-level verification.

## Interpreting Results

- Higher `completeness` means broader answer coverage.
- Lower `hallucination_rate` is better; this is the most important safety metric.
- `citation_accuracy` and `hallucination_rate` are linked but not identical:
  - `citation_accuracy` focuses on supported claims.
  - `hallucination_rate` focuses on unsupported claims.
- For unanswerable tasks, a good model should avoid fabrication and increase `uncertainty_reported`.

## File Conventions

- Harness output: `YYYYMMDDTHHMMSSZ.json`.
- Standard logging schema template: `result_template.json`.
- Human summary: generated with `python -m eval.summarize_results --latest`.

## Recommended Workflow

1. Run eval: `python -m eval.harness`
2. Summarize latest: `python -m eval.summarize_results --latest`
3. Paste/commit key summary into README when metrics materially change.
